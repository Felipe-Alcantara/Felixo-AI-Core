# Relatório: 2026-04-30

Terceiro dia de atividade do repositório. O trabalho consolidou continuidade de CLIs, processos persistentes, arquitetura híbrida com MCP, adapters persistentes fake/ACP/app-server, opções runtime de modelo e a tasklist de UX/workspace de 30/04/2026.

## Implementações

### CLI, threads e persistência

- Corrigida contagem/contexto determinístico no Codex.

- Separados Terminal Adapters e planejamento de execução.

- Adicionados fake persistent agents e tipos `AgentEvent`.
- Implementado adapter Gemini ACP.

- Renderer passou a reconhecer tipos persistentes de CLI.

### Opções runtime de modelo

- Adicionado suporte backend a opções runtime de modelo.
- Composer ganhou seletores de modelo do provedor e effort.
- Criado plano de confirmações/permissões para CLI.

### Limpeza operacional

- `start_app.py` passou a limpar processos antigos antes do launch.
- README foi atualizado.

### Chat e contexto

- "Novo chat" limpa input, anexos, terminal local e estado de streaming.
- A thread corrente é enviada ao backend por `cli:reset-thread` para encerrar sessão persistente anterior quando houver.
- Anexos do botão `+` no composer entram como contexto da próxima mensagem.
- Arquivos de texto pequenos recebem preview textual truncado; binários entram como metadados.

### Projetos

- Projetos cadastrados persistem via `project-storage.ts`.
- IDs de projetos ativos também persistem e são validados contra a lista atual.
- Sidebar exibe projetos ativos mesmo com a seção colapsada.

### Automações

- Criado catálogo de automações padrão em `data/automations.ts`.
- Criado storage de automações personalizadas em `automation-storage.ts`.
- `AutomationsModal` lista automações padrão/customizadas, cria customizadas e aplica prompts no composer.

### Modelos e Felixo

- Clicar em um modelo na sidebar seleciona o modelo e abre `ModelSettingsModal`.
- O modal de modelos mostra capacidades por CLI e campos configuráveis (`providerModel`, `reasoningEffort`).
- O botão "Felixo" abre `FelixoSettingsModal`, separado das configurações de modelos.

### Code/Git

- Criado `CodePanel` para consulta inicial de Git.
- Criado IPC `git:get-summary`.
- Criado `git-service.cjs` com comandos allowlisted e read-only:
  - `status --short --branch`;
  - `diff --stat`;
  - `log -5 --oneline --decorate=short`;
  - `branch --show-current`.
- Adicionados testes unitários para parsing e allowlist Git.

### Terminal e QA Logger

- Terminal lateral alterna entre `Threads` e `Orquestrador`.
- Terminal pode ser recolhido e redimensionado horizontalmente.
- QA Logger pode ser recolhido e redimensionado verticalmente.
- QA Logger preserva assinatura de eventos mesmo recolhido.

### Interface

- Janela principal Electron configurada explicitamente como redimensionável, maximizável e fullscreen.
- Selects nativos receberam tema escuro para corrigir fundo branco/texto branco.
- Placeholder visual da tela inicial foi substituído pela logo do portfólio em `/brand/felixo-logo.png`.

## Testes de integração real (CLIs)

### Codex app-server

Protocolo JSON-RPC 2.0 testado com `codex app-server` real (v0.125.0):

- `initialize` → `{userAgent, codexHome, ...}` ✓
- `initialized` notification → sem resposta ✓
- `thread/start` → `{thread: {id, ...}}` ✓
- `thread/started` notification ✓
- `turn/start` → `turn/started` + `item/agentMessage/delta` + `turn/completed` ✓
- Auto-aprovação de `item/commandExecution/requestApproval` ✓

### Gemini ACP (v1 — `gemini 0.40.1`)

O protocolo ACP mudou do esquema antigo (`newSession`, `prompt`) para o ACP SDK v1.
Protocolo testado com `gemini --acp` real:

- `initialize({protocolVersion:1})` → `{agentCapabilities, authMethods, ...}` ✓
- `authenticate({methodId:'oauth-personal'})` → `{}` ✓
- `session/new({cwd, mcpServers:[]})` → `{sessionId, ...}` ✓
- `session/prompt({sessionId, prompt:[...]})` → `session/update` chunks + `{stopReason:'end_turn'}` ✓
- `session/update` notificação com `agent_message_chunk` ✓

O `gemini-acp-adapter.cjs` foi atualizado para o protocolo v1 e os testes unitários foram reescritos.
120 testes passando após a atualização.
