# Relatório: 2026-04-29

Status: concluido.

## Resumo

Segundo dia de atividade do repositório. O trabalho saiu do protótipo visual para um backend Electron capaz de executar CLIs em streaming, adicionou adapters para Claude/Codex/Gemini, painel de QA, busca de chats, projetos Git, reorganização completa da documentação e primeira versão do terminal bruto/persistência de threads.

Fonte: commits de 2026-04-29 no Git.

## Implementações

### Modelos e fluxo de configuração

- Refatorado o fluxo de configuração de modelos locais.
- Adicionados contratos de stream para modelos.
- O prompt passou a incluir metadados do modelo para melhorar contexto da CLI.

### Backend CLI em streaming

- Criado backend Electron para execução de CLIs com eventos em streaming.
- Adicionados adapters para Claude, Codex e Gemini.
- Criados serviços de processo, leitura JSONL e guard de stdout.
- Corrigidos casos de Codex fora de repositório confiável, stdout com blanks iniciais e prompts não JSON do Gemini.
- Adicionado QA Logger no backend e painel correspondente no frontend.

### Chat e resposta em tempo real

- O chat passou a renderizar respostas em streaming.
- O workspace passou a enviar mensagens para o backend Electron.
- Foram ajustados estados de resposta, erro e conclusão.

### Sidebar, busca e projetos

- Removida seção de recentes da sidebar.
- Sidebar ganhou redimensionamento por drag.
- Adicionada busca em tempo real sobre histórico de sessões.
- Criado modal de projetos com importação de repositório único ou workspace.
- Corrigidos botões fora do Electron, fechamento por overlay e seleção nativa de pasta.
- Lista de projetos ficou expansível, com projeto ativo sempre visível.
- Projetos ativos passaram a ser múltiplos e entrar no prompt com diff de adição/remoção.

### Terminal e continuidade

- Criado plano dedicado de terminal persistente.
- Adicionado painel de terminal bruto em tempo real.
- Threads de terminal passaram a persistir por conversa.
- Codex começou a retomar threads quando metadados de sessão existem.
- Corrigido fechamento de `stdin` em execuções one-shot.

### Documentação

- Separado README e ROADMAP.
- Centralizados documentos Markdown em `docs/`.
- Adicionadas pastas e documentos de arquitetura, backend, frontend, projeto e padrões.
- Atualizado roadmap com frente de terminal persistente.
