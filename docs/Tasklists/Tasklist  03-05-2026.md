# Felixo AI Core — Tasklist de Portabilidade, Build e Releases

> **Objetivo:** adaptar o Felixo AI Core para funcionar tanto rodando pelo código-fonte quanto como aplicativo instalável/baixável para diferentes sistemas operacionais, com geração automática de releases sempre que houver push na branch `production`.

---

## 0. Visão Geral

O Felixo AI Core deve suportar dois modos principais de uso:

1. **Modo desenvolvimento / código-fonte**

   * Usuário clona o repositório.
   * Instala dependências.
   * Executa o app localmente.
   * Usa o sistema para desenvolvimento, testes e contribuição open source.

2. **Modo aplicativo empacotado / release**

   * Usuário baixa um executável ou instalador.
   * Instala ou executa o app sem precisar mexer no código-fonte.
   * Usa o app como produto final.

O código precisa ser adaptativo para diferentes sistemas operacionais, considerando diferenças de caminhos, shells, permissões, instalação, binários, comandos de terminal, variáveis de ambiente e formatos de empacotamento.

## 20. Observação final para o agente

Não implementar a release automática antes de estabilizar o modo empacotado localmente.

A ordem mais segura é:

1. fazer o app rodar bem pelo código-fonte;
2. adaptar paths, shell e detecção de CLIs por SO;
3. gerar pacote local funcional;
4. criar CI básica;
5. criar release automático em `production`;
6. expandir para todos os sistemas operacionais.

A automação só deve publicar releases depois que o build local estiver minimamente confiável.


O executável deve ser auto-suficiente, ou seja, não deve depender de nenhuma instalação externa feita pelo usuário, mas sim do que é necessário para o funcionamento do app que deve estar incluso no build, e caso tenha uma atualização que necessite de alguma instalação externa, o app deve notificar o usuário e guiá-lo para fazer a instalação, o app deve ser auto atualizavel

O app também deve ser auto detectavel no sistema (Ex: WinRAR, 7-Zip, etc), ou seja, quando o usuário clicar em algum arquivo do tipo "Felixo AI Core", ele deve abrir o app
